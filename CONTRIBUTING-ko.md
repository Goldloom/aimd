# AIMD 기여 안내

AIMD에 기여해주셔서 감사합니다.

이 저장소의 현재 공개 범위는 아래와 같습니다.

- v1.5까지의 레거시 공개 스펙
- 공개 AIMD Core v2 초안
- 예시, 문서, validator 관점 자료

상위 상용 레이어는 별도로 존재할 수 있습니다. 이 저장소의 기여는 공개 범위 안에 머무르는 것을 원칙으로 합니다.

---

## 기여하기 좋은 영역

다음과 같은 기여를 권장합니다.

- 공개 Core 스펙의 명확화
- AIMD Core 예시 문서 추가
- validator용 테스트 케이스 보강
- 문서 개선
- 번역 수정

반대로, 비공개 또는 상용 전용 AIMD 레이어를 이 저장소의 공개 범위로 전제하는 변경은 피하는 것이 좋습니다.

---

## 기여 원칙

- 공개 Core는 작게 유지합니다.
- 과한 prose보다 의미 명확성을 우선합니다.
- 암묵적 의미보다 명시적 구조를 우선합니다.
- 강한 상호운용성 근거가 없으면 public Core 범위를 넓히지 않습니다.
- 레거시 v1.5는 참조 이력으로 취급하고 미래 기본 방향으로 간주하지 않습니다.

---

## 실무 흐름

1. 공개 스펙 의미에 영향을 주는 변경이라면 먼저 이슈나 토론을 엽니다.
2. 예시나 문법 가이드를 바꾼다면 `spec/en/AIMD-core-v2.md`와 정합성을 유지합니다.
3. 한국어 문서를 추가하거나 수정할 때는 영문 원문과 의미가 맞도록 유지합니다.
4. 예시는 canonical 하고 validator 친화적으로 유지합니다.
5. 변경 이유를 간단히 설명한 pull request를 제출합니다.

---

## 스펙 변경 제안 시

Core 스펙 변경을 제안할 때는 아래를 함께 포함해 주세요.

- 해결하려는 문제
- 왜 이것이 상위 레이어가 아니라 public Core에 들어가야 하는지
- 변경 전/후 예시
- validator에 미치는 영향

기본 원칙은 보수적 진화입니다. 공개 Core는 천천히 바뀌어야 합니다.
